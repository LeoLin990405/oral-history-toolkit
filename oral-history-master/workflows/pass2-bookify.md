# 工作流：pass2-bookify（Pass 2 书面化 · 发表级编辑）

> ⚠ **前置硬条件**：只能在 **Pass 1 整理稿已经受访人审阅签字** 之后运行。
> 未签字的整理稿不得进入 Pass 2。这是口述史伦理底线。

## Pass 1 vs Pass 2 的根本区别

| | Pass 1 去口语化（oral-history-master 主流程） | Pass 2 书面化（本工作流） |
|---|---|---|
| 输入 | 录音转写稿 | **已签字的** Pass 1 整理稿 |
| 目标 | 忠实、可读、可签字 | 发表级：流畅、有结构、合期刊体例 |
| 可否调整篇章 | ❌ 不重排（仅文末提示时间线） | ✅ 可按主题/时序重组 |
| 可否提炼删节 | ❌ 不删实质内容 | ✅ 可删冗余、合并重复主题（但保留关键史实与原意） |
| 口语质感 | 大量保留 | 转为书面语体 |
| 不变的底线 | 不捏造 | **仍不捏造**；引文/史实/受访人原意不可改 |

## 流程（与 Pass 1 复用同一项目，新开 `pass=2`）

1. 复制签字稿为 Pass 2 输入：`output/整理稿.md` → `output/书面稿_draft.md`，manifest 标 `pass: 2`。
2. 读 `references/methodology.md` §2（两道工序），明确"现在允许重组，但仍不可捏造"。
3. 书面化处理（主 agent，逐节）：
   - 口语连接、口头禅 → 书面衔接
   - 同主题分散段落 → 按主题/时序聚合（**记录重排映射**，可回溯到签字稿）
   - 冗余、重复主题 → 精炼（保留关键史实、人物、判断）
   - 补注释/小标题（整理者注用脚注或 `〔整理者注：…〕`，与受访人原话区分）
4. **史实守门**：人名/机构/专名沿用 Pass 1 的 `term_lock.md`；引文逐字核对签字稿；
   受访人的判断与情感不得在"书面化"名义下被中性化。
5. 产出 `output/书面稿.md` + `review/Pass2重排映射.md`（书面稿每节 ← 来自签字稿哪几段）。
6. 仍建议受访人/版权方复核书面稿（发表前）。

## 红线
- 凡 Pass 2 中"为通顺"而改动的史实、引文、因果判断，一律视为越界。
- 受访人在 Pass 1 保留的矛盾/不确定，Pass 2 可在脚注说明，但**不得擅自"裁决"**哪个版本为真。
