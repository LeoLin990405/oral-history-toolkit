# 口述史整理方法论（蒸馏自 11 部文献）

> 这是**定尺度的依据**，不是逐条执行的清单。Editor 的执行尺度看 `examples.md` 的范例；
> 本文件回答"为什么这么改 / 这么留"。Surveyor 与 quality-guard 必读，Editor 选读。

## 0. 一句话立场

**录音才是史料；整理稿是对录音的"忠实转写 + 最小干预"。** 整理者的笔越轻越好——
理想状态是读者读整理稿，听到的仍是受访人本人的声音，而不是整理者的文风。

## 1. 思想谱系（这套规则从哪来）

| 来源 | 给我们的核心约束 |
|---|---|
| **熊卫民**（科学家口述史，中科大科学史一脉） | 科学家口述的"存真优先"；整理稿须经受访人审阅；科技体制术语（室/所/院/学部/课题组）原样保留 |
| **张藜**（老科学家学术成长资料采集工程 / 中国科学家博物馆） | 采集—整理—入库的规范；多重材料（录音/手稿/档案）互证；人名机构必须可核 |
| **王扬宗**（中科院科学史） | 科学事件、机构沿革的史实严谨性；不可用"通顺"掩盖事实模糊 |
| **Donald Ritchie,《Doing Oral History》** | 转录与编辑的职业伦理；区分"清理"（去噪）与"改写"（越界）；署名与授权 |
| **Alessandro Portelli,《The Death of Luigi Trastulli》** | **口误、矛盾、不确定本身是史料**——它们暴露记忆与心理的真实，不可被"修正"抹平 |
| Paul Thompson《The Voice of the Past》/ Valerie Yow 等 | 受访人主体性；整理者的"在场"要透明（概述用第三人称标注） |

> Portelli 的洞见对本工具最关键：**"错"往往比"对"更真**。受访人把 1958 说成 1959 又自我纠正、
> 前后说法矛盾、反复说"记不清"——这些不是要被熨平的褶皱，而是记忆的纹理。我们**保留并标注**，
> 绝不替他做一个看起来很顺、其实失真的版本。

## 2. 存真 vs 可读：两道工序

口述史整理是**两道分开的工序**，本工具（oral-history-master）只做第一道：

- **Pass 1 · 去口语化（本工具）** → 产出"整理稿"：去口语噪音、纠显性口误与转写错、
  补最小语法、统一专名。**保留声音、逻辑、判断、情感、不确定性。** 用途：受访人审阅签字、存档。
- **Pass 2 · 书面化（另一流程 `workflows/pass2-bookify.md`）** → 在**已签字的整理稿**上做发表级编辑：
  调整篇章、提炼、套期刊体例。**只能在 Pass 1 定稿后做，不能混在一起**，否则两头不到岸。

冲突时的优先级（贯穿全程）：**保护(A) > 转换(C) > 语境(D) > 清除(B)**。宁可多留，不可误删。

## 3. A / B / C / D 四类（原则，非清单）

### A 类 · 保护（绝不动）
口述史的"不可让渡区"。改了 A 类 = 篡改，不是整理。
1. **核心叙事逻辑**：因果、前后顺序、主观判断——受访人怎么理解事件，就怎么留。
2. **语义性情感标记**：说实话 / 其实 / 我真的 / 你想想——是功能词，承载态度，删了就变味。
3. **方言 / 行话 / 职业黑话**：如"吃皇粮""跑口子"，保留原貌（必要时加引号），这是身份与时代的印记。
4. **前后矛盾**：保留 + `⚠`，交人工 / 供研究，**绝不替受访人圆场**（Portelli）。
5. **个性化表达**：独有措辞、比喻、讲故事的方式——标准化掉就抹掉了人。
6. **科技体制术语**：室 / 所 / 院 / 学部 / 课题组 / 863 / 学部委员……原样，这是科技史的硬通货。
7. **不确定表述**："大概""记不清""好像"——记忆的真实质感，删了就是伪造确定性。

### B 类 · 清除（口语噪音，静默删）
不承载任何信息的口语残渣。删，且**不逐个标注**（否则满屏标记没法读）。
- 填充词：嗯 / 啊 / 呃 / 哦；非指代的"那个"（"那个，我就…"删；"那个方案"留）；非引用的"就是说""怎么说呢"
- 无修辞功能的逐字重复："特别特别"取一个；但"一年又一年"是修辞 → 留
- 采访人的倾听反馈：嗯 / 对 / 是吗 / 然后呢（**采访人的实质提问要留**）
- 流程寒暄："咱们开始吧""今天就到这儿"

### C 类 · 转换（口语→书面，保守，照范例尺度）
- **口误自纠**：取受访人最终确认的版本，`〔标注原口误〕`（保留"他纠正过"这个事实）
- **口语连接词**：反正→总之、完了→之后、（无时序的）然后→随后/接着；**有时序义的"然后"保留**
- **中文口语语法**：删冗余"有"（他有来过→他来过）、给他说→跟他说、有够累→很累
- **补全**：**只补语法成分**（缺主语 / 连词），**绝不补充信息**。拿不准就不补，标 `⚠`
- **时间线**：默认不重排；叙事跳跃时文末附"建议时间线"，**正文不动**

### D 类 · 语境（由 edit_spec 开关决定）
- 采访人话语：保留【问】/ 移脚注 / 删除
- 方言：保留原文 / 标准化 + 原文括注
- 补全力度：最小补全 / 仅标注不补（高仿真场景）

## 4. 整理者的三条职业纪律（Ritchie）

1. **不可捏造**：缺的主语、连词可补；受访人没说的信息一个字不能加。
2. **改动可追溯**：意义级改动留 `〔说明〕`；真正的可追溯靠"原始转写 ⟷ 整理稿"对照（对照稿），
   **不靠编造"改了 N 处"统计**。
3. **授权与署名**：整理稿须经受访人审阅签字；无录音 / 原始材料对照的转写，关键处保留 `⚠` 提醒核查。

## 5. 为什么"范例驱动"而不是"规则驱动"

24 条规则并排成清单，LLM 处理长文时会漂移——应用一部分、漏一部分。**2~3 段完整的
before/after 范例能把规则隐式编码**，模型照着仿写的稳定性远高于逐条比对。
所以：Editor 的执行尺度以 `examples.md` 为准，本方法论提供"为什么"，范例提供"怎么改到什么程度"。
