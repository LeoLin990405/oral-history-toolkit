> 本文件回答"为什么这么改 / 这么留"。Surveyor 与 quality-guard 必读，Editor 选读。

> 与本工具的 A/B/C/D 执行体系对齐。框架概念归属各原始文献作者（见下表）。

## 0. 一句话立场

**录音才是史料；整理稿是对录音的"忠实转写 + 最小干预"。** 整理者的笔越轻越好——
理想状态是读者读整理稿，听到的仍是受访人本人的声音，而不是整理者的文风。

## 2. 三层认识论（口述史为什么不同于普通编辑）

1. **认识论层**：口述史追问的是"记忆之真"逼近"历史之真"，本就有失真。整理者的工作是**检验、修订、补充、取舍呈现**，而非消灭失真。
2. **媒介层**：口语→文字必然丢信息（语调、语速、停顿、肢体）。转录稿是录音的"诠释"，不是"等价物"。
3. **伦理层**：口述史是访谈者与受访人的**共同产品**，定稿须受访人审阅签字；整理者不能把自己的历史认识强加给受访人。

## 3. 双本制 · 生熟光谱 · 本工具定位

口述史料从"生"到"熟"是连续光谱，不是两极跳跃：

```
原始录音(最"生") → 逐字转录稿 → Pass1 去口语化稿 → Pass2 书面整理稿 → 出版访谈录(最"熟")
                                    └────── 本工具处理范围 ──────┘
```

| | **档案本**（原始抄本） | **传播本**（编辑抄本） |
|---|---|---|
| 定位 | 录音的另存，服务研究与存档 | 公开面世形式 |
| 原则 | 最大限度如实抄录 | 方便读者准确理解 |
| 对应 | 逐字转录 → **Pass 1 整理稿** | Pass 1 → **Pass 2 书面稿** |

本工具的 **Pass 1（去口语化）= 从原始稿到初整稿**；**Pass 2（书面化，见 `workflows/pass2-bookify.md`）= 从初整稿到可发表整理稿**，须在签字后另起流程。

## 4. 红线（R1–R11，操作底线）

**保真**：R1 不删改实质 · R2 不添原文没有的事实/观点/数据/引文 · R3 不改立场判断情感 · R4 "记不清/大概"照录不代编 · R5 矛盾叙述均保留、整理人不当裁判。
**透明**：R6 改动可追溯 · R7 整理者增补必须标记区分（括号/脚注/编者注）· R8 标注是否经受访人审阅。
**伦理**：R9 不删改不添加不虚夸不炒作 · R10 不把整理者认识强加给受访人 · R11 共同产品，署"某某口述，某某访谈整理"。
**可逆性（方法论底线）**：所有编辑必须能回溯到原始录音/逐字稿；若某操作使原文无法还原，则不可做。

## 5. A / B / C / D 执行体系（把红线落成可操作的尺度）

冲突时优先级：**A 保护 > C 转换 > D 语境 > B 清除**（宁可多留，不可误删）。

### A 类 · 保护（绝不动）— 对应 R1–R5
核心叙事逻辑 / 功能性情感词（说实话·其实·你想想）/ 方言行话（如"吃皇粮"，保留+必要时引号，**不靠变形拼写标方言**）/ 前后矛盾（保留+`⚠`，不圆场，Portelli）/ 个性化表达与金句 / 科技体制术语（室·所·院·学部·课题组·863）/ 不确定表述（大概·记不清·好像）。

### B 类 · 清除（口语噪音，静默删，不逐个标）
填充词（嗯·啊·呃；非指代"那个"；非引用"就是说""怎么说呢"）/ 无修辞重复（"特别特别"取一；"笑死人了羞死人了"这类有递进的保留）/ 采访人倾听反馈（嗯·对·是吗）/ 流程寒暄。
> 中华规范补充：仅删"无实际意义的口头语"；有语义功能的口头表达必须留。

### C 类 · 转换（保守，照范例尺度）
口误自纠取最终值并`〔标注〕`/ "错误话头"删中断的无信息开端、保留实际表达 / 口语连接词（反正→总之，无时序"然后"→随后；有时序"然后"保留）/ 中文口语语法（他有来过→他来过）/ **补全只补语法成分，绝不补信息**，拿不准标`⚠` / 时间线不自动重排，跳跃时文末提示。
> 中文语助词（呢/吧/嘛）：有语法功能（疑问/推测/强调）保留，纯填充才删——比英文 uh/um 更需谨慎。

### D 类 · 语境（开关决定）
采访人话语（保留【问】/脚注/删除）/ 方言（保留/标准化+原文括注）/ 补全力度（最小补全/仅标注不补）。

### 声音 / 姿态标注（A 类的特例，必须保）
（笑）（哭）（长叹）（拍桌）等——这些在文字里可能承载反讽等关键信息，删了会造成严重误会。分辨不清用 `[…]` 或 `【录音不清】`。

## 6. 为什么"范例驱动"而不是"规则驱动"

把一长串规则并排成清单，LLM 处理长文时会漂移——应用一部分、漏一部分；规则越堆越长，反而越"粗"。
**2~3 段（越多越好）高质量的 before/after 范例能把规则隐式编码**，模型照着仿写的稳定性远高于逐条比对。
所以：Editor 的执行尺度以 `examples.md` 为准，本方法论提供"为什么"，范例提供"改到什么程度"。
**持续改进的杠杆是补真实范例对，不是堆更多规则**（见 `examples.md` 末节与 `CONTRIBUTING-methodology.md`）。
