# 方法论贡献指南 · 用你们自己的语料持续改进这个 skill

> 分工：**口述史研究者出方法论与范例，工程持续把它接进流水线。** 这份指南规定
> "蒸馏产物长什么样、塞进哪里"，让你（或拆书 skill）的输出能**直接命中格式、零返工**。

## 0. 一条铁律：范例 > 规则

这个 skill 的质量天花板 = `references/examples.md` 里 before/after 范例对的质量，**不是规则数量**。
"规则太粗"翻车的根因就是**蒸馏太长 → 规则堆砌**。所以：**规则精炼几条到十几条就够，重心放在补真实范例对。**

## 1. 你要产出的三样"参考文本"（及落点）

| 产物 | 落入 | 要求 |
|---|---|---|
| **A. 精炼整理规则** | `references/methodology.md`（§5 A/B/C/D 下补充，或新增条目） | 每条一句话 + 出处；**保持 tight，别堆长** |
| **B. before/after 范例对** | `references/examples.md`（最高杠杆） | 每条 = 【原始转写】+【整理稿】+ 一句改动逻辑 |
| **C. 书面化规则/风格** | `workflows/pass2-bookify.md` | 句式/结构/用词/脚注/体例，归 Pass 2 |

> ❌ 不要引入"删除 XX 处"之类的**改动数量统计**——LLM 数不准自己的编辑，那是假可追溯。可追溯靠对照稿。

## 2. 工作流（对接你现有的拆书 / book-distillation skill）

```
Phase 1 选材  5 本经典 + 论文/规范 + 已发表访谈稿
   （建议：张藜、中华规范、Ritchie、陈墨、牛津手册、熊卫民、《中国科技史杂志》…）
        ↓
Phase 2 蒸馏  用你的拆书 skill 分块蒸馏（fit 上下文）
   ⚠ 目标产物对准 §1 的 A/B/C 格式，不是泛泛读书笔记
        ↓
Phase 3 装配  规则→methodology.md ｜ 范例对→examples.md ｜ 书面化→pass2-bookify.md
        ↓
Phase 4 验证  held-out 测试：拿"已发表访谈稿"对应的原始口语喂 skill，
   比对 skill 输出 vs 学者发表版 —— 差距即下一轮改进方向（最硬的验证）
        ↓
Phase 5 迭代  按差距补范例 / 微调规则 / 调开关 → 回 Phase 4
```

## 3. 给拆书 skill 的蒸馏指令模板（让输出直接命中格式）

> 把下面贴给你的拆书 skill，对每本/每篇文献跑一遍，产出能直接装配的内容：

```
你在为一个"口述史去口语化整理"工具蒸馏方法论。读完本文献，只产出三样：

A. 整理规则（最多 10 条）：每条一句话 + (本文献出处)。只要能落到操作的，
   删冗余原则、空泛表态一律不要。

B. before/after 范例对（尽量多，越具体越好）：从文献的"记录稿/整理稿对照实例"
   里抽，格式——
   【原始转写】……（口语原貌）
   【整理稿】……（去口语化后；意义级改动用〔说明〕，待核对用 ⚠）
   改动逻辑：一句话
   文献没有现成对照实例时，不要编造，留空。

C. 书面化要点（仅当文献涉及"书面化/发表体例"）：句式/结构/用词/脚注/风格，
   每条一句 + 出处。

不要写综述、不要客套、不要统计字数。
```

## 4. 红线

- 范例对必须真实配对（原始口语 ↔ 整理/发表稿）；拿不到原始口语的发表稿，只当"风格锚"，标明非完整范例对。
- 验证用 held-out（没进过蒸馏的）发表稿，避免自我印证。
- **涉密语料只在本地处理**；公开仓库/issue 里只放脱敏或合成片段。
- 规则与原作者出处一并记，便于回查与署名（方法论谱系见 `references/methodology.md` §1）。
