# Oral History Quality Guard（忠实度质量门）

> 口述史的交付标准不是"读起来顺"，
> 而是"对得起录音"。本技能是**交付前的最后一道硬门**：抱着"这稿一定有问题"的
> 对抗心态去找茬，找不到才放行。

## 何时运行
`oral-history-master` Step 7。`review/对照稿.md` 已生成，整理稿待交付。

## 必读
```
oral-history-master/references/shared-standards.md   # 伦理红线 + 标记约定
oral-history-master/references/methodology.md        # A 类保护 / Portelli 矛盾观
```

## 审计材料
- `review/对照稿.md`（原始转写 ⟷ 整理稿，逐块）—— **主审材料**
- `term_lock.md`（术语表 + 语气画像 + 红线）—— 对照基准
- `review/{一致性报告,忠实度报告,待核对清单}.md` —— 机检结果，作为线索而非结论

## 七维对抗式审计（逐块过）

对**每一块**，拿整理稿和原始正文逐句比，专门找下面七类违规：

1. **捏造（最致命）** — 整理稿里有没有原文根本没有的信息、人名、年份、因果？
   逐句问："这句的每个信息点，原文都有出处吗？"
2. **伪造确定性** — 原文的"大概/记不清/好像"被改成肯定语气了吗？
3. **圆场矛盾** — 原文前后矛盾的地方，被整理成一个"自洽版本"了吗？（应保留 + ⚠）
4. **抹方言/个性** — 行话、特有比喻、叙述习惯被标准化成教科书腔了吗？
5. **过度补全** — "补语法"是否越界变成了"补内容"？（对照 fidelity 报告里偏长的块重点查）
6. **⚠ 漏标** — 人名/机构/专名/可疑同音字，该标 ⚠ 而没标的？
7. **术语不统一** — 同一专名全篇写法一致吗？（核对 term_lock + 一致性报告）

> 额外查：A 类功能性情感词（说实话/其实/你想想）是否被当噪音误删；采访人实质提问是否按开关处理。

## 判定与产出

逐块给判定，写入 `review/QA报告.md`：
```
## chunk_NNN
判定：PASS / NEEDS FIX
- [若 NEEDS FIX] 第 X 维违规：原文「…」→ 整理稿「…」 问题：捏造了"…" 建议：删除/改回/补 ⚠
```

汇总：
```
# QA 报告
总块数 N｜PASS M｜NEEDS FIX K
最严重问题（按 1→7 优先级）：……
结论：☐ 通过，可交付　☐ 不通过，返工块：chunk_AAA, chunk_BBB …
```

## 门规则
- 任何一块 **NEEDS FIX** → 整稿不通过。把问题块退回 `oral-history-master` Editor 修，重跑 Step 5–6，再回本门复审。
- 全 PASS 才放行。交付时必须明确陈述：**已对照原文逐句审忠实度，已查七维违规，已核 ⚠ 与术语一致性。**
- 放行后仍提醒：**此稿须经受访人审阅签字方可使用。**

## 对抗心态（重要）
不要走过场。默认"整理者可能在某处不知不觉添了字 / 把矛盾抹平了 / 把方言改顺了"，
带着怀疑去翻每一块。宁可多挑几个"疑似"交人工，也不要漏掉一个真捏造——
口述史里，一处捏造就能毁掉整篇的史料价值。
