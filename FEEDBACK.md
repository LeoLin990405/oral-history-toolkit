# 反馈 · FEEDBACK

> 这条通道就是为"**你们用、你们测、我来改**"准备的。
> 尽管用力测——报错、整理得不对、想加功能，都欢迎提。我会尽快跟进修。

## ⚠ 先看这条（涉密）

**千万不要把任何真实访谈内容贴进公开 issue。** 本仓库是公开的。

要举例说明问题，请用 **编造/脱敏的片段**，或**只描述现象**（如"方言被改成书面语了""一个人名在第 3 段和第 8 段写法不一样"）。涉及具体内容的，直接私信我即可。

## 两种反馈方式（任选）

1. **GitHub Issue**（推荐，可追踪）：仓库页面点 **Issues → New issue**，选下面的模板填。
2. **直接私信我**（微信等）：内容一样，怕麻烦就这样。

## 反馈类型 & 需要的信息

| 类型 | 请提供 |
|---|---|
| 🐛 **脚本报错 / 装不上 / 跑不动** | 哪一步、完整报错信息、系统（mac/win/linux）、`python3 --version` |
| 📝 **整理质量不对** | 哪类问题（删过头 / 添了原文没有的内容 / 方言被改 / 矛盾被圆 / 术语不统一 / `⚠` 该标没标）+ **脱敏或编造的**例子 |
| 🎚 **方法论 / 开关建议** | 你想改哪条规则、或想要什么仿真度/开关行为 |
| ✨ **想要的新功能** | 想解决什么场景；目前怎么绕过的 |

## 懒人模板（复制填空）

```
类型：报错 / 质量 / 方法论 / 功能
现象：
（质量类用脱敏或编造的例子，别贴真实内容）
期望：
环境（报错才填）：系统___ / Python 版本___ / 卡在哪一步___
```
